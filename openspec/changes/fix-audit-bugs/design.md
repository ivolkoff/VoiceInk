## Context

Аудит ([CODE_AUDIT.md](../../../CODE_AUDIT.md)) дал 28 находок, уже прошедших адверсариальную верификацию сабагентами. Однако находки фиксируют состояние кода на момент аудита и привязаны к номерам строк, которые могли сдвинуться. Исправления точечные и пересекают несколько модулей (Transcription, Services, PowerMode, Shortcuts, Views), общего архитектурного рефакторинга не требуют.

## Goals / Non-Goals

**Goals:**
- Устранить 28 подтверждённых дефектов поэтапно: Critical → High → Medium → Low.
- Перед каждым фиксом **повторно подтвердить, что баг существует в текущем коде** (строки могли сдвинуться, баг мог быть исправлен в другом change).
- Минимальные, локальные изменения; сохранить существующий стиль и поведение вне дефекта.

**Non-Goals:**
- Архитектурный рефакторинг, новые фичи, изменение публичного API.
- Исправление pre-existing build-ошибок (whisper, Sparkle, code signing), не связанных с находками.
- Удаление чужого dead-code, не относящегося к находкам.

## Decisions

**Решение: каждый таск начинается с шага верификации «валидный баг или нет».**
Находки привязаны к строкам и могли устареть. Перед фиксом — открыть файл, найти конструкцию по описанию (не по номеру строки), подтвердить дефект. Если баг отсутствует/уже исправлен — таск помечается как N/A с пометкой почему, фикс не применяется. Альтернатива (слепо чинить по строкам) отвергнута: риск сломать корректный код или промахнуться мимо сдвинувшейся строки.

**Решение: группировка по приоритету, а не по модулю.**
Critical/High краши и потеря данных — первыми. Это позволяет остановиться на любом этапе с максимальной пользой. Альтернатива (по модулю) хуже приоритизирует риск для пользователя.

**Решение: единый паттерн на категорию.**
- Crash → `guard let`/`as?`/`.first`/проверка `isEmpty`.
- Concurrency → `await MainActor.run` или изоляция актором; повторять существующий паттерн в том же файле.
- Error-handling → do/catch с логом (копировать стиль уже корректного места рядом, напр. `VoiceInkEngine.swift:252`).
- Memory → `deinit { NotificationCenter.default.removeObserver(...) }`.
- State → сброс флага/полей во всех ветвях завершения.

**Решение: верификация фикса — `xcodebuild` + точечная проверка по сценарию из spec.**
Где возможно — отметить, что end-to-end проверка требует запуска в Xcode (как в существующих change'ах репозитория).

## Risks / Trade-offs

- [Строки в находках сдвинулись] → искать по описанию конструкции/grep, не по номеру строки; шаг верификации в каждом таске.
- [Concurrency-фикс меняет тайминг и вносит регрессию] → повторять паттерн уже корректных путей в том же файле; не менять логику, только изоляцию.
- [Часть находок — ложные срабатывания, прошедшие верификацию] → шаг «валидный или баг» отсекает их перед фиксом; помечать N/A с обоснованием.
- [Build не собирается локально из-за pre-existing ошибок] → отделять новые ошибки компиляции от pre-existing (whisper/Sparkle/signing).
